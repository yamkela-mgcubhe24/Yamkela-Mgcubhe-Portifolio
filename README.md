### Task
My task was to learn how Markdown works and use it to structure and present my CV in a professional and readable format on GitHub. I needed to ensure that all sections such as personal details, education, skills, and experience were clearly organised.

### Action
Since I was clueless about Markdown and its syntax, I had to do my research and practice using features such as headings, bullet points, and text formatting. I then converted my existing CV into Markdown by carefully restructuring the content and ensuring it was clean and easy to read. I also previewed my work on GitHub to check formatting and made adjustments where necessary.

### Result
In the end, I successfully created a well-structured CV using Markdown on GitHub. This process improved my technical skills and helped me understand how developers present content in a simple but professional way. I also gained confidence in using GitHub as a tool for showcasing my work.
